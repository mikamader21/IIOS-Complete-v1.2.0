# IIOS Implementation Roadmap

**Version:** 1.2.0

## Phase 0 — Ratify Foundation

Approve Charter/Constitution, resolve Critical audit findings, create private repos and set Owner decisions.

## Phase 1 — Local engineering baseline

Reproducible tooling, lint/test, CI, secret scanning, Docker Compose local dependencies and typed schemas.

## Phase 2 — Knowledge foundation

Open independent Obsidian Vault, metadata/templates, backup/restore test, source register and Graphify POC with soft guidance.

## Phase 3 — Governance core

Action classifier, deterministic policy evaluator, approval state machine, scoped capability design, audit/cost schemas and kill interface.

## Phase 4 — Hermes laboratory

Pinned release, isolated profile, Docker terminal backend, no forwarded secrets, context tests, one safe skill and telemetry.

## Phase 5 — Hybrid private infrastructure

VPS, private network, database, backups, staging, monitoring and incident runbook.

## Phase 6 — Control Center MVP

Health, tasks, approvals, audit, costs, knowledge, runtime and security modules.

## Phase 7 — First read-only domain

FundingPips/prop observability: connector, rules/drawdown, calendar, alerts and reconciliation. No order endpoints.

## Phase 8 — Controlled intelligence

Add Research, Audit/Risk or Developer profile only after measurable need.

## Phase 9 — Skill catalog

Promote proven procedures through tests/security/cost review. Generated skills remain lab until approved.

## Phase 10 — Production hardening

Separated roles, protected audit, egress, SBOM, recovery exercise, SLOs, budget alerts and staged upgrades.

## Phase 11 — Additional domains

Each domain needs threat model, data contract, permissions, cost budget, acceptance tests and ADR.
