# IIOS Operating Model: Claude, Hermes, Obsidian and Graphify

**Version:** 1.2.0

## Role statement

**Fable 5 reasons at the frontier; Sonnet 5 implements economically; Claude Code builds; Cowork performs supervised knowledge work; Obsidian conserves; Graphify relates; Hermes operates persistently; Governance controls; Owner decides.**

## Claude Fable 5

Use for exceptional architecture, complex root-cause analysis, high-stakes reviews, novel research synthesis and final audit of critical changes.

Do not use as default router target, memory system, policy engine or automatic approver. Its safeguards may route some requests to another model; workflows must tolerate model substitution and validate outputs.

## Claude Sonnet 5

Preferred day-to-day model for coding, tool use, research and implementation while benchmarks and budget support it. Use effort levels and validation appropriate to task risk.

## Claude Code

Primary engineering interface. Reads project context, edits files, runs tests, supports hooks, permissions, skills and subagents. It builds the system but does not govern itself.

## Claude Cowork

Use for supervised documents, reports, research synthesis, spreadsheets, presentations and working with authorized folders/apps. Outputs enter IIOS only after provenance and review.

## Hermes Agent

Persistent runtime for CLI/Desktop/gateway, profiles, sessions, cron, skills, subagents and remote operation. Run local for lab and on private VPS for continuous tasks.

## Obsidian

Human-first institutional knowledge surface based on local Markdown. It is the second brain for reviewed knowledge, not the transactional database or secret store.

## Graphify

Derived knowledge graph for code and documents. Use soft graph-first guidance: orient with the graph, verify with sources. Do not block source reading in MVP.

## Escalation pattern

```text
No LLM → low-cost model → Sonnet 5 → Fable 5 → multi-model review
```

Escalate from complexity, uncertainty, risk or failed validation—not prestige.

## Multi-agent policy

Start single-session. Add subagents for isolated parallel research. Add persistent teams only after duplicate-work, coordination, token and shutdown controls are measured.
